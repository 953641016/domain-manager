## 1. 角色定义

| 角色 | 标识 | Web 登录 | 说明 |
|------|------|---------|------|
| 业务人员 | `business` | ❌ | 通过飞书发起域名/DNS 申请 |
| 域名专员 | `domain_spec` | ✅ | 管理分配给自己的域名账号，审批业务人员申请 |
| 系统管理员 | `admin` | ✅ | 管理人员（用户），**不涉及域名账号** |
| 超级管理员 | `super_admin` | ✅ | 唯一，拥有所有权限，负责授权确认 |

> **职责分离原则**：  
> - `admin` 管人，不管账号  
> - `domain_spec` 管账号，不管人  
> - `super_admin` 是两条授权链的终点

## 3. 飞书授权确认流程

### 3.1 两类授权发起场景

#### 场景 A：域名专员 / 系统管理员发起操作

```
发起人（专员/管理员）在 Web 后台操作
        ↓
系统创建 Confirmation 记录（status: pending）
        ↓
系统向超级管理员发送飞书授权卡片
  ┌────────────────────────────┐
  │ 🔐 账号配置授权申请         │
  │ 操作人：张三（域名专员）     │
  │ 操作类型：新增注册账号       │
  │ 账号名：Cloudflare-主账号   │
  │ API Key：****5678           │
  │  [✅ 授权执行]  [❌ 拒绝]   │
  └────────────────────────────┘
        ↓ 超管点击
系统执行实际操作（写库）
        ↓
通知发起人：操作结果（成功 / 拒绝原因）
```

#### 场景 B：超级管理员发起操作（自确认）

超级管理员自己发起的操作同样需要飞书确认，防止误操作并留存审计。

```
超管在 Web 后台操作
        ↓
系统创建 Confirmation 记录（status: pending）
        ↓
系统向超管自身发送飞书确认卡片
  ┌────────────────────────────┐
  │ 🔑 操作确认（请确认您发起的操作）│
  │ 操作类型：删除用户            │
  │ 目标用户：李四（域名专员）     │
  │  [✅ 确认执行]  [❌ 取消]    │
  └────────────────────────────┘
        ↓ 超管在飞书点击确认
系统执行实际操作
        ↓
审计日志记录操作结果
```

### 3.2 确认记录字段说明

| 字段 | 说明 |
|------|------|
| `operation_type` | 操作类型（见下方枚举） |
| `initiator_user_id` | 发起人 |
| `requires_super_admin` | 是否需要超管确认（所有需确认操作均为 true） |
| `operation_details` | 操作载荷（JSON，待执行的数据） |
| `status` | pending / approved / rejected / cancelled |
| `feishu_message_id` | 绑定飞书卡片消息，防重复处理 |
| `confirmed_at` | 确认时间 |

### 3.3 操作类型枚举

```
域名账号类：
  add_reg_account      新增注册账号
  update_reg_account   修改注册账号
  delete_reg_account   删除注册账号
  add_dns_account      新增解析账号
  update_dns_account   修改解析账号
  delete_dns_account   删除解析账号
  set_default_config   设置默认配置

人员管理类：
  add_user             添加用户
  update_user          修改用户信息
  update_user_role     变更用户角色
  delete_user          删除/禁用用户
  set_specialist       设置归属专员
```

### 3.4 确认超时处理

| 规则 | 值 |
|------|----|
| 确认有效期 | 24 小时 |
| 超时自动作废 | 状态变为 `cancelled` |
| 超时通知 | 通知发起人重新提交 |

## 4. 数据隔离规则

### 4.1 域名专员视角

- **只能看到** `owner_id = 自己` 的注册账号、解析账号、域名
- **只能审批** `assigned_specialist_id = 自己` 的业务人员申请
- 无法访问其他专员的任何数据

### 4.2 系统管理员视角

- 可以看到**所有用户**，但**看不到**注册账号、解析账号
- 可以看到**所有申请记录**（用于管理追踪）
- 无法访问域名列表中的账号信息

### 4.3 超级管理员视角

- 可以看到所有数据，无隔离限制

### 4.4 业务人员→专员路由

```
业务人员（business）
  └── assigned_specialist_id  →  域名专员
        ↑ 由 admin 或 super_admin 在用户管理中设置
```

业务人员提交申请时：
- 系统自动查找 `requester.assigned_specialist_id`
- 将审批通知发送给对应专员
- 若未设置归属专员，发给所有专员或系统管理员

## 5. 人员添加与权限设置流程

### 5.1 添加业务人员（推荐：扫码添加）

```
管理员在后台点击"扫码添加"
        ↓
生成飞书 OAuth 二维码
        ↓
新员工用飞书扫码
        ↓
系统自动获取飞书用户信息
        ↓
创建用户（默认角色：business）
        ↓
管理员设置归属专员（assigned_specialist_id）⚠️ 需超管确认
```

### 5.2 添加域名专员

```
管理员（admin）在后台创建用户
  └─ 角色选择：domain_spec
  └─ 飞书用户 ID：填写或搜索
        ↓ 提交
系统向超管发飞书确认卡片
        ↓ 超管确认
用户创建成功，通知管理员
```

### 5.3 变更用户角色

所有角色变更均需超管飞书确认，记录变更前后的角色值到审计日志。

### 5.4 禁用/删除用户

- **禁用**（推荐）：保留数据，仅阻止登录
- **删除**：高危操作，需超管确认，保留申请记录（软删除）
