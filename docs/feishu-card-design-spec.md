# 飞书卡片消息设计规范

> 版本：v1.0  
> 建立日期：2026-05-31  
> 适用范围：`feishu_service.py` / `user_confirmation_service.py` / `execution_service.py` 中所有飞书卡片消息  
> 原则：运营视角，风险可辨、信息可读、操作可追

---

## 1. 颜色语义（`header.template`）

| 颜色 | 场景 | 示例 |
|------|------|------|
| `red` | 高危/不可逆操作、申请被拒绝、超管转让 | 删除用户、删除账号、删除服务商、超管转让、❌ 拒绝通知 |
| `orange` | 普通授权申请（可恢复/可撤销）、执行失败 | 创建/修改用户、账号配置、⚠️ 失败通知 |
| `yellow` | 非紧急提醒 | 域名到期 > 30 天 |
| `green` | 成功结果通知 | ✅ 操作执行成功 |
| `blue` | 中性技术操作（DNS） | DNS 解析申请 |

**判断规则（优先级从高到低）：**

```python
if 操作不可逆（delete_*、超管转让）:
    color = "red"
elif 操作为授权申请（create/update/activate/deactivate）:
    color = "orange"
elif 域名到期提醒:
    color = "red"   # expire_days <= 7
    color = "orange"  # expire_days <= 30
    color = "yellow"  # expire_days > 30
elif 结果通知:
    color = "green"   # 成功
    color = "red"     # 被拒绝
    color = "orange"  # 执行失败
```

## 2. 卡片结构模板

### 2.1 授权申请卡片（发给超管）

```
┌──────────────────────────────────────────┐
│ [颜色] Emoji + 操作类别授权申请             │  ← header
├──────────────────────────────────────────┤
│ **申请人**：张健                           │
│ **操作事项**：修改用户 ⚠️（不可恢复）       │  ← 高危操作追加此后缀
│ **操作对象**：张立坤（系统管理员）           │
├──────────────────────────────────────────┤  ← hr（仅当有变更明细时）
│ **变更内容**：                             │
│ · 角色 → 域名专员                          │
│ · 部门 → 技术部                            │
├──────────────────────────────────────────┤  ← hr
│ [✅ 授权执行]          [❌ 拒绝]           │  ← action
├──────────────────────────────────────────┤
│ ⏱ 申请有效期 24 小时，逾期自动作废          │  ← note
└──────────────────────────────────────────┘
```

**字段渲染规则：**
- 所有标签名加粗：`**申请人**`、`**操作事项**`、`**操作对象**`、`**变更内容**`
- 变更明细每条以 `· ` 开头，`字段名 → 新值`
- 空值字段不展示（过滤掉 `None` 和 `""`）
- 角色用中文名（`super_admin` → `超级管理员`，通过 `_role_name()` 转换）
- 布尔值用中文（`is_active=True` → `启用`，`False` → `禁用`）

### 2.2 结果通知卡片（发给发起人）

```
┌──────────────────────────────────────────┐
│ [green/red/orange] ✅/❌/⚠️ 操作执行成功  │
├──────────────────────────────────────────┤
│ **操作**：创建用户 - 张立坤               │
│ **拒绝原因**：xxx（仅拒绝时展示）          │
│ **原因**：xxx（仅失败时展示）              │
└──────────────────────────────────────────┘
```

### 2.3 域名到期提醒卡片

```

│ [red/orange/yellow] 🚨/⚠️/📅 域名到期提醒 │

│ **到期日期**：2026-06-15（若有则展示）     │
│ **注册商**：GoDaddy（若有则展示）          │
│                                          │
│ **仅剩 5 天**，请立即联系注册商续费！       │  ← 紧急度文案见下方
└──────────────────────────────────────────┘
```

```
┌──────────────────────────────────────────┐
│ [blue] 🌐 DNS 解析申请                    │
├──────────────────────────────────────────┤
│ **申请人**：张健                           │
│ **域名**：example.com                     │
│ **解析平台**：Cloudflare                   │
│ **记录数**：3 条                           │
├──────────────────────────────────────────┤
│ • **api.example.com**  A  →  1.2.3.4     │
│ • **www.example.com**  CNAME  →  ...     │
├──────────────────────────────────────────┤
│ [✅ 批准执行]      [❌ 拒绝申请]           │
├──────────────────────────────────────────┤
│ 申请编号：#a1b2c3d4                       │  ← note，取 UUID 前 8 位
└──────────────────────────────────────────┘
```

## 3. 卡片标题规范

| 场景 | 标题 |
|------|------|
| 超管转让 | `👑 超管转让授权申请` |
| 创建/修改/禁用/激活用户 | `👤 用户管理授权申请` |
| 删除用户 | `👤 用户管理授权申请`（红色卡片区分） |
| 新增/修改/删除注册账号/解析账号 | `🔐 账号配置授权申请` |
| 新增/修改/删除服务商 | `🏷️ 服务商配置授权申请` |
| DNS 解析申请 | `🌐 DNS 解析申请` |
| 域名到期（≤7天） | `🚨 域名即将到期` |
| 域名到期（≤30天） | `⚠️ 域名到期提醒` |
| 域名到期（>30天） | `📅 域名到期提醒` |
| 操作执行成功 | `✅ 操作执行成功` |
| 操作申请被拒绝 | `❌ 操作申请被拒绝` |
| 操作执行失败 | `⚠️ 操作执行失败` |

## 4. 按钮文案规范

| 按钮类型 | 文案 | type |
|---------|------|------|
| 批准（用户/账号类） | `✅ 授权执行` | `primary` |
| 批准（DNS 类） | `✅ 批准执行` | `primary` |
| 拒绝 | `❌ 拒绝申请` | `danger` |

> ❌ 不要只写"拒绝"——语义不清，"拒绝申请"更明确。

---

## 5. 风险标注规范

高危操作（不可逆）在**操作事项**后追加警示：

```python
danger_note = {
    "delete_user":        " ⚠️（不可恢复）",
    "delete_reg_account": " ⚠️（不可恢复）",
    "delete_dns_account": " ⚠️（不可恢复）",
    "delete_provider":    " ⚠️（不可恢复）",
}
```

超管转让卡片整体用红色，不再追加后缀（卡片标题和内容已足够明确）。

---

## 6. 域名到期提醒文案规范

| 剩余天数 | 标题 | 颜色 | 正文紧急度文案 |
|---------|------|------|--------------|
| ≤ 7 天 | `🚨 域名即将到期` | `red` | `**仅剩 {n} 天**，请立即联系注册商续费！` |
| ≤ 30 天 | `⚠️ 域名到期提醒` | `orange` | `剩余 **{n}** 天，请尽快安排续费。` |
| > 30 天 | `📅 域名到期提醒` | `yellow` | `剩余 {n} 天，请适时安排续费。` |

---

## 7. 通用配置项

所有卡片必须加宽屏配置：

```python
"config": {"wide_screen_mode": True}
```

---

## 8. 底部注脚（`note` 元素）规范

| 卡片类型 | 注脚内容 |
|---------|---------|
| 所有授权申请卡片 | `⏱ 申请有效期 24 小时，逾期自动作废` |
| DNS 审批卡片 | `申请编号：#XXXXXXXX`（UUID 前 8 位） |
| 结果通知卡片 | 无注脚（正文已足够） |
| 到期提醒卡片 | 无注脚 |
