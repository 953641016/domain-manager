## 4. Docker服务管理



## 5. SSL证书管理

### 5.1 证书信息

- **证书颁发机构**: Let's Encrypt
- **域名**: d.fwxg.com
- **有效期**: 90天（自动续期）
- **到期日期**: 2026-08-28
- **证书位置**: /etc/letsencrypt/live/d.fwxg.com/

### 5.2 证书申请

首次申请（已完成）：

```bash
# 使用webroot验证方式
certbot certonly --webroot -w /opt/domain-manager/letsencrypt/webroot -d d.fwxg.com --email admin@fwxg.com --agree-tos --non-interactive
```

### 5.3 证书续期

Certbot已配置自动续期任务，也可以手动续期：

```bash
# 检查续期
certbot renew --dry-run

# 手动续期
certbot renew

# 续期后需要复制证书到项目目录并重启Nginx
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cd /opt/domain-manager
docker restart domain-manager-nginx
```

### 5.4 查看证书信息

```bash
# 查看证书详细信息
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -text -noout

# 查看证书到期时间
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates
```

## 6. 应用更新部署流程

> **当前方案：服务器构建，手动部署**
> 零额外成本，流程简单，适合现阶段单服务器小团队。

### 6.1 本地提交代码

```bash
git add .
git commit -m "描述改动"
git push origin main
```

### 6.2 部署到服务器（每次发布时执行）

```bash
# 1. SSH 连接服务器
ssh -i C:\Users\张健\.toolchain\zj_deploy.pem root@115.28.211.155

# 2. 进入项目目录，拉取最新代码
cd /opt/domain-manager
git pull

# 3. 重建并重启容器
docker compose up -d --build

# 4. 确认服务正常
docker compose ps
curl -s http://localhost:8000/health
```

> **构建时间说明**（Docker 层缓存机制）：
> - 日常改业务代码 → `npm install` 跳过 → 只跑 `vite build`，约 **30-60 秒**
> - 修改了 `package.json`（新增依赖）→ 重跑 `npm install`，约 **3-5 分钟**（偶发）

### 6.3 只部署前端或后端

如果只改了其中一个，可以只重建那个服务，更快：

```bash
# 只重建前端
docker compose up -d --build --no-deps frontend

# 只重建后端
docker compose up -d --build --no-deps backend
```

### 6.4 回滚

```bash
# 回退到上一个版本
cd /opt/domain-manager
git log --oneline -5          # 找到目标提交
git reset --hard <commit>     # 回退代码
docker compose up -d --build  # 重建
```

### 6.5 SSH 私钥权限（Windows 常见坑）

Windows 下若私钥文件对其他用户可读，OpenSSH 会报 `UNPROTECTED PRIVATE KEY FILE` 拒绝连接：

```powershell
# 收紧权限（只需执行一次）
icacls "C:\Users\张健\.toolchain\zj_deploy.pem" /inheritance:r
icacls "C:\Users\张健\.toolchain\zj_deploy.pem" /grant:r "$env:USERNAME:R"
```

### 6.6 历史记录（已完成的一次性配置）

- 服务器目录原为 scp 部署，2026-05-31 原地转为 Git 检出（跟踪 `origin/main`）。
- 数据卷 `domain-manager_backend_data`（含 SQLite）、`.env`、`letsencrypt/` 均保留。
- `origin` 远程地址内嵌 GitHub PAT，建议后续改用 Deploy Key 并轮换 PAT。

### 6.7 未来升级路径（按需，现阶段不必做）

| 触发条件 | 升级方案 |
|---------|---------|
| 部署频繁（>10次/天），手动 SSH 麻烦 | 加 GitHub Actions 自动 SSH 触发 |

| 团队超过 3 人 | 云效 / 自建 Jenkins |

## 8. 常见问题排查

### 8.1 服务无法访问

```bash
# 1. 检查容器状态
docker ps

# 2. 检查服务日志
docker logs domain-manager-nginx --tail 50
docker logs domain-manager-backend --tail 50

# 3. 检查端口监听
netstat -tlnp | grep -E '80|443|8000|3000'
```

### 8.2 证书问题

```bash
# 检查证书是否存在
ls -la /opt/domain-manager/letsencrypt/live/d.fwxg.com/

# 检查证书有效期
openssl x509 -in /opt/domain-manager/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates

# 重新复制证书
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
docker restart domain-manager-nginx
```

### 8.3 前端或API访问404

```bash
# 检查Nginx配置
docker exec domain-manager-nginx cat /etc/nginx/conf.d/domainmanager.conf

# 测试上游服务
docker exec domain-manager-nginx wget -O - http://frontend:80/dm/ --spider
docker exec domain-manager-nginx wget -O - http://backend:8000/health --spider

# 重启Nginx
docker restart domain-manager-nginx
```

```bash

tail -f /var/log/nginx/access.log  # 在宿主机或容器中

docker logs domain-manager-backend -f --tail 100
```

### 9.3 防火墙配置

确保只开放必要的端口：

```bash
# 查看防火墙状态（Ubuntu使用ufw）
ufw status

# 应该只开放：
# - 22 (SSH)
# - 80 (HTTP)
# - 443 (HTTPS)
```
