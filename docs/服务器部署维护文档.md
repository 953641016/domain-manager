## 2. SSH 连接

### 2.1 密钥说明

| 文件 | 说明 |
|------|------|
| `C:\Users\张健\.toolchain\zj_deploy.pem` | **正式使用版**（权限已收紧，OpenSSH 可识别） |
| 仓库根 `zj.pem` | 权限过宽，OpenSSH 会报 UNPROTECTED PRIVATE KEY FILE，**不可直接用** |

### 2.2 SSH 命令

**PowerShell / CMD：**
```powershell
ssh -i "C:\Users\张健\.toolchain\zj_deploy.pem" -o StrictHostKeyChecking=no root@115.28.211.155
```

**Git Bash / WSL（路径写法不同）：**
```bash
# Git Bash（C: → /c）
ssh -i "/c/Users/张健/.toolchain/zj_deploy.pem" -o StrictHostKeyChecking=no root@115.28.211.155

# WSL（需先 cp 到 /tmp 绕过中文路径）
cp "/mnt/c/Users/张健/.toolchain/zj_deploy.pem" /tmp/zj_deploy.pem
chmod 600 /tmp/zj_deploy.pem
ssh -i /tmp/zj_deploy.pem -o StrictHostKeyChecking=no root@115.28.211.155
```

> **AI 助手注意**：在 Bash 工具中执行 SSH 时，`C:/Users/张健/` 路径因含中文可能无法加载密钥，
> 必须先将密钥复制到 `/tmp/zj_deploy.pem` 再执行。

### 2.3 私钥权限修复（仅需执行一次）

若 OpenSSH 报 `UNPROTECTED PRIVATE KEY FILE`：

```powershell
icacls "C:\Users\张健\.toolchain\zj_deploy.pem" /inheritance:r
icacls "C:\Users\张健\.toolchain\zj_deploy.pem" /grant:r "$env:USERNAME:R"
```

## 3. 日常代码部署

### 3.1 标准流程

```
本地改代码 → git commit → git push origin main → SSH 服务器执行部署脚本
```

#### 步骤一：本地提交

```bash
git add <文件>
git commit -m "描述改动"
git push origin main
```

#### 步骤二：服务器执行部署（一条命令）

```bash
# 只改了后端（最常见，约 30-60 秒）
ssh -i /tmp/zj_deploy.pem root@115.28.211.155 "bash /opt/domain-manager/deploy.sh backend"

# 只改了前端（约 30-60 秒）
ssh -i /tmp/zj_deploy.pem root@115.28.211.155 "bash /opt/domain-manager/deploy.sh frontend"

# 全量重建（约 3-5 分钟）
ssh -i /tmp/zj_deploy.pem root@115.28.211.155 "bash /opt/domain-manager/deploy.sh"
```

脚本内部执行：`git pull` → `docker compose up -d --build <service>` → 清理旧镜像 → 验证状态。

> **构建时间参考**：
> - 日常改业务代码（无新依赖）：约 **30-60 秒**（Docker 层缓存命中）
> - 新增 pip/npm 依赖：约 **3-5 分钟**

### 3.2 回滚

```bash
ssh -i /tmp/zj_deploy.pem root@115.28.211.155 "
  cd /opt/domain-manager &&
  git log --oneline -5 &&
  git reset --hard <目标commit> &&
  bash deploy.sh
"
```

## 6. SSL 证书管理

- **证书机构**：Let's Encrypt（90 天有效期，已配置 certbot 自动续期）
- **到期日**：2026-08-28
- **证书位置**：服务器 `/etc/letsencrypt/live/d.fwxg.com/`

```bash
# 查看证书到期时间
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates

# 手动续期
certbot renew

# 续期后同步到项目目录并重启 Nginx
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
docker restart domain-manager-nginx
```

## 8. 故障排查

### 服务无法访问

```bash
# 1. 容器状态
docker ps

# 2. 查日志
docker logs domain-manager-nginx --tail 50
docker logs domain-manager-backend --tail 50

# 3. 端口监听
netstat -tlnp | grep -E '80|443|8000|3000'
```

### 前端/API 404

```bash
# 检查 Nginx 配置
docker exec domain-manager-nginx cat /etc/nginx/conf.d/domainmanager.conf

# 测试上游服务
docker exec domain-manager-nginx wget -qO- http://frontend:80/dm/ --spider
docker exec domain-manager-nginx wget -qO- http://backend:8000/health --spider

# 重载 Nginx
docker restart domain-manager-nginx
```

### SSL 证书问题

```bash
# 证书是否存在
ls -la /opt/domain-manager/letsencrypt/live/d.fwxg.com/

# 有效期
openssl x509 -in /opt/domain-manager/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates
```
