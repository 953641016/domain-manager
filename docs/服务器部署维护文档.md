## 5. SSL证书管理

### 5.1 证书信息

- **证书颁发机构**: Let's Encrypt
- **域名**: d.fwxg.com
- **有效期**: 90天（自动续期）
- **到期日期**: 2026-08-28
- **证书位置**: /etc/letsencrypt/live/d.fwxg.com/

### 5.2 证书申请

首次申请（已完成）：

```bash
# 使用webroot验证方式
certbot certonly --webroot -w /opt/domain-manager/letsencrypt/webroot -d d.fwxg.com --email admin@fwxg.com --agree-tos --non-interactive
```

### 5.3 证书续期

Certbot已配置自动续期任务，也可以手动续期：

```bash
# 检查续期
certbot renew --dry-run

# 手动续期
certbot renew

# 续期后需要复制证书到项目目录并重启Nginx
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cd /opt/domain-manager
docker restart domain-manager-nginx
```

### 5.4 查看证书信息

```bash
# 查看证书详细信息
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -text -noout

# 查看证书到期时间
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates
```

## 6. 应用更新部署流程

> **2026-05-31 起改为 Git 部署**：服务器 `/opt/domain-manager` 已是一个 Git 检出，
> 跟踪 `origin/main`。此后更新统一走 `git pull` + `docker compose up -d --build`，
> 不再用 scp/tar 拷贝。

### 6.1 本地开发与提交

```bash
# 1. 在本地进行代码修改

# 2. 提交代码到 main 并推送
git add .
git commit -m "更新描述"
git push origin main
```

> 提示：前端构建在容器内完成（frontend/Dockerfile 跑 `npm install && npm run build`），
> 本地无需安装 Node 即可部署；如需本地校验，运行 `cd frontend && npm install && npm run build`。

### 6.2 服务器端更新（标准流程）

```bash
# 1. 连接服务器（注意：Windows 下私钥权限过宽会被 OpenSSH 拒绝，见 2.3）
ssh -i /path/to/zj.pem root@115.28.211.155

# 2. 进入项目目录并拉取最新代码
cd /opt/domain-manager
git pull

# 3. 重新构建并启动（构建失败时旧容器继续运行，不会停服）
docker compose up -d --build

# 4. 验证
docker compose ps                              # 三个容器均 Up，backend 显示 healthy
curl -s http://localhost:8000/health           # 期望 {"status":"ok"}
curl -s -o /dev/null -w '%{http_code}\n' -k https://localhost/dm/   # 期望 200
docker compose logs -f backend                 # 观察启动/执行日志
```

### 6.3 回滚

```bash
# 方式一：回退到上一个提交并重建
cd /opt/domain-manager
git log --oneline -5          # 找到要回退的提交
git reset --hard <commit>
docker compose up -d --build

# 方式二：用首次转 Git 前的全量备份恢复
#   备份文件：/opt/dm-backup-pre-git.tar.gz
#   后端 .env 备份：/opt/dm-backend.env.bak
```

### 6.4 Git 部署的一次性配置说明（已完成，仅作记录）

- 服务器目录原为 scp 部署、非 Git 仓库。2026-05-31 在 `/opt/domain-manager` 原地执行
  `git init` → 添加 `origin` → `git fetch` → `git reset --hard origin/main` 转为 Git 检出。
- 目录名保持 `domain-manager` 不变，故 `domain-manager_backend_data` 数据卷（含 SQLite）
  以及 `.env`、`letsencrypt/` 等被忽略/未跟踪的运行时文件均原样保留。
- **凭证**：`origin` 远程地址内嵌了 GitHub PAT（私有仓库 pull 需要）。该 PAT 存在
  `/opt/domain-manager/.git/config` 中，**建议后续改用只读 Deploy Key（SSH）并轮换该 PAT**。

### 6.5 SSH 私钥权限（Windows 常见坑）

Windows 下若 `zj.pem` 对 `Authenticated Users`/`Users` 可读，OpenSSH 会报
`UNPROTECTED PRIVATE KEY FILE` 并拒绝连接。把私钥放到仅本人可读的位置即可：

```powershell
icacls "C:\path\to\zj.pem" /inheritance:r
icacls "C:\path\to\zj.pem" /grant:r "$env:USERNAME:R"
```

## 8. 常见问题排查

### 8.1 服务无法访问

```bash
# 1. 检查容器状态
docker ps

# 2. 检查服务日志
docker logs domain-manager-nginx --tail 50
docker logs domain-manager-backend --tail 50

# 3. 检查端口监听
netstat -tlnp | grep -E '80|443|8000|3000'
```

### 8.2 证书问题

```bash
# 检查证书是否存在
ls -la /opt/domain-manager/letsencrypt/live/d.fwxg.com/

# 检查证书有效期
openssl x509 -in /opt/domain-manager/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates

# 重新复制证书
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
docker restart domain-manager-nginx
```

### 8.3 前端或API访问404

```bash
# 检查Nginx配置
docker exec domain-manager-nginx cat /etc/nginx/conf.d/domainmanager.conf

# 测试上游服务
docker exec domain-manager-nginx wget -O - http://frontend:80/dm/ --spider
docker exec domain-manager-nginx wget -O - http://backend:8000/health --spider

# 重启Nginx
docker restart domain-manager-nginx
```
