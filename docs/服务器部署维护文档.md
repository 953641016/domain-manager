## 4. Docker服务管理

### 4.1 启动服务

```bash
cd /opt/domain-manager
docker compose up -d
```

### 4.2 停止服务

```bash
cd /opt/domain-manager
docker compose down
```

### 4.3 重启服务

```bash
cd /opt/domain-manager
docker compose restart
```

### 4.4 查看服务状态

```bash
docker ps
# 或者
cd /opt/domain-manager
docker compose ps
```

### 4.5 查看服务日志

```bash
# 查看所有服务日志
cd /opt/domain-manager
docker compose logs -f

# 查看单个服务日志
docker logs domain-manager-backend -f
docker logs domain-manager-frontend -f
docker logs domain-manager-nginx -f
```

### 4.6 重新构建并启动

```bash
cd /opt/domain-manager
docker compose up -d --build
```

## 5. SSL证书管理

### 5.1 证书信息

- **证书颁发机构**: Let's Encrypt
- **域名**: d.fwxg.com
- **有效期**: 90天（自动续期）
- **到期日期**: 2026-08-28
- **证书位置**: /etc/letsencrypt/live/d.fwxg.com/

### 5.2 证书申请

首次申请（已完成）：

```bash
# 使用webroot验证方式
certbot certonly --webroot -w /opt/domain-manager/letsencrypt/webroot -d d.fwxg.com --email admin@fwxg.com --agree-tos --non-interactive
```

### 5.3 证书续期

Certbot已配置自动续期任务，也可以手动续期：

```bash
# 检查续期
certbot renew --dry-run

# 手动续期
certbot renew

# 续期后需要复制证书到项目目录并重启Nginx
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cd /opt/domain-manager
docker restart domain-manager-nginx
```

### 5.4 查看证书信息

```bash
# 查看证书详细信息
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -text -noout

# 查看证书到期时间
openssl x509 -in /etc/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates
```

## 6. 应用更新部署流程

### 6.1 本地开发与提交

```bash
# 1. 在本地进行代码修改

# 2. 提交代码到Git
git add .
git commit -m "更新描述"
git push
```

### 6.2 服务器端更新

```bash
# 1. 连接服务器
ssh -i /path/to/zj.pem root@115.28.211.155

# 2. 进入项目目录
cd /opt/domain-manager

# 如果是Git仓库（可选）
git pull

# 3. 重新构建并启动服务
docker compose down
docker compose up -d --build

# 4. 查看服务状态
docker ps
docker compose logs -f
```

### 6.3 快速更新（仅文件修改）

如果只是修改配置文件或部分代码：

```bash
# 1. 复制修改的文件到服务器
scp -i /path/to/zj.pem local/file root@115.28.211.155:/opt/domain-manager/path/

# 2. 重启相关服务
docker restart domain-manager-nginx  # 仅修改Nginx配置
# 或者
docker compose restart  # 重启所有服务
```

## 8. 常见问题排查

### 8.1 服务无法访问

```bash
# 1. 检查容器状态
docker ps

# 2. 检查服务日志
docker logs domain-manager-nginx --tail 50
docker logs domain-manager-backend --tail 50

# 3. 检查端口监听
netstat -tlnp | grep -E '80|443|8000|3000'
```

### 8.2 证书问题

```bash
# 检查证书是否存在
ls -la /opt/domain-manager/letsencrypt/live/d.fwxg.com/

# 检查证书有效期
openssl x509 -in /opt/domain-manager/letsencrypt/live/d.fwxg.com/fullchain.pem -noout -dates

# 重新复制证书
cp -L /etc/letsencrypt/live/d.fwxg.com/fullchain.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
cp -L /etc/letsencrypt/live/d.fwxg.com/privkey.pem /opt/domain-manager/letsencrypt/live/d.fwxg.com/
docker restart domain-manager-nginx
```

### 8.3 前端或API访问404

```bash
# 检查Nginx配置
docker exec domain-manager-nginx cat /etc/nginx/conf.d/domainmanager.conf

# 测试上游服务
docker exec domain-manager-nginx wget -O - http://frontend:80/dm/ --spider
docker exec domain-manager-nginx wget -O - http://backend:8000/health --spider

# 重启Nginx
docker restart domain-manager-nginx
```

## 9. 安全建议

### 9.1 定期更新

```bash
# 更新系统包
apt update
apt upgrade -y

# 定期更新Docker镜像
docker compose pull
docker compose up -d
```

### 9.2 日志监控

```bash
# 监控Nginx访问日志
tail -f /var/log/nginx/access.log  # 在宿主机或容器中

# 监控应用日志
docker logs domain-manager-backend -f --tail 100
```

### 9.3 防火墙配置

确保只开放必要的端口：

```bash
# 查看防火墙状态（Ubuntu使用ufw）
ufw status

# 应该只开放：
# - 22 (SSH)
# - 80 (HTTP)
# - 443 (HTTPS)
```
