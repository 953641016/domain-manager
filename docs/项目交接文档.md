## 项目概述

### 项目简介

域名管家是一个企业级域名管理自动化平台，通过飞书机器人提供便捷的域名注册和DNS解析申请审批流程，配备Web管理后台进行记录查看、报表统计和日志分析。

### 核心设计理念

- **两层架构分离**:
  - 业务层: 飞书驱动（申请、审批、执行）
  - 配置层: Web后台（账号配置、记录查询、统计分析）

- **安全边界**:
  - Web后台不参与业务流程推进
  - 敏感操作需经超级管理员飞书确认

### 主要功能

| 功能模块 | 状态 |
|---------|------|
| 飞书OAuth登录 | ✅ |
| 域名管理 | ✅ |
| DNS解析管理 | ✅ |
| 申请审批流程 | ✅ |
| 注册商/DNS适配器 | ✅ |
| 审计日志 | ✅ |
| 定时任务 | ✅ |
| SSL证书管理 | ✅（项目自身域名的SSL，非业务域名） |
| 飞书机器人 | ✅（基础框架） |
| 飞书文档按钮集成 | ✅（已有完整方案设计，后端部分已实现） |
| 飞书多维表格 | ✅（已有完整方案设计，通过按钮读取记录） |

### 10个核心模型

| DnsRecord | `models/dns.py` | DNS解析记录、记录类型、TTL等 |
| Request | `models/request.py` | 申请记录（域名注册/DNS解析） |
| RegistrarAccount | `models/domain.py` | 注册商账号、加密存储API密钥 |
| DnsAccount | `models/dns.py` | DNS服务商账号、加密存储API密钥 |
| Provider | `models/domain.py` | 注册商/DNS服务商信息 |

| FeishuBitable | `models/feishu_bitable.py` | 飞书多维表格配置（备用） |

### 数据库设计要点

- **敏感字段加密**: 使用 Fernet 对称加密存储 API 密钥
- **定时任务到期检查**: 每天 09:00 检查域名到期
- **用户归属关系**: business 用户必须归属唯一的 domain_spec
- **权限硬边界**: admin 无法查看/操作域名业务数据（can_view_domains=False）及服务商账号；但可访问审计日志和SSL证书管理（系统运维范围）

### 关键配置表

`models/system.py` 提供系统默认配置：
- 默认注册商
- 默认DNS服务商
- 默认账号
- 其他全局配置项

## 角色与权限

### 四大角色

| 角色 | 标识 | Web登录 |
|------|------|---------|
| 业务人员 | `business` | ❌ |
| 域名专员 | `domain_spec` | ✅ |
| 系统管理员 | `admin` | ✅ |
| 超级管理员 | `super_admin` | ✅ |

### 权限矩阵（核心设计原则）

**两条硬边界**:
1. **admin 不碰业务**: 管理员只看到「人员管理 + 全局记录/日志/统计」
2. **domain_spec 不碰人**: 域名专员只看到「自己的账号/服务商/域名/申请」

### 权限表

完整权限定义见:
- 代码: `backend/app/models/permission.py`
- 文档: `docs/权限与流程设计.md`

| 操作 | business | domain_spec | admin | super_admin |
|------|:--------:|:-----------:|:-----:|:-----------:|
| 提交域名注册申请（飞书） | ✅ | ✅ | ❌ | ✅ |
| 直接注册/解析（免审批） | ❌ | ✅ | ❌ | ✅ |
| 审批申请 | ❌ | ✅（归属自己的） | ❌ | ✅ |
| 查看域名列表/详情/DNS | ❌ | ✅（自己的） | ❌ | ✅ |
| 管理注册账号 | ❌ | ✅（需超管确认） | ❌ | ✅（需超管确认） |
| 人员管理 | ❌ | ❌ | ✅（需超管确认） | ✅（需超管确认） |

### 需要超管确认的操作

所有敏感操作需经超级管理员飞书卡片确认：
- 新增/修改/删除注册/DNS账号
- 新增/修改/删除服务商
- 设置默认配置

### 飞书文档按钮集成方案（已确认）

**文档位置**: `docs/feishu-doc-integration.md`

**方案要点**:
- **飞书文档是主申请入口**，飞书机器人为备用入口
- **文档表格纯输入，无状态追踪** —— 表格只记录"我想要的配置"
- **所有容错在后端** —— 幂等判断、重复检测、可用性检查均由服务端负责
- **按钮类型**: 飞书文档"打开超链接"按钮，点击打开我们的确认页
- **按钮URL格式**: `https://{our-domain}/feishu/confirm-request?section=vercel`

**完整触发链路**:
```
用户在 Bitable 填好 DNS 记录
    ↓
点击文档中的"申请XXX"按钮
    ↓
浏览器打开确认页（需飞书 OAuth 登录）
    ↓
用户填写域名 + 点"确认提交"
    ↓
POST /api/v1/feishu/submit-request
    ↓
读取 Bitable 所有记录 + 创建申请记录
    ↓
向归属专员发飞书审批卡片
    ↓
专员点"批准" → 执行 → 差异化通知
```

**已实现部分（根据文档）**:
- 读取 Bitable 记录: `feishu_service.read_bitable_records()`
- 向专员发 DNS 审批卡片: `feishu_service.send_dns_approval_card()`
- 主确认页数据接口: `GET /feishu/confirm-data`
- Bitable 自助绑定: `POST /feishu/bind-bitable`
- 提交申请: `POST /feishu/submit-request`
- 确认页前端: `frontend/src/pages/FeishuConfirm.tsx`

### 飞书机器人命令（当前实现）

当前 `bots/feishu.py` 实现的命令:

## 部署指南

### 环境变量配置

必须配置的环境变量见: `backend/.env.example`

| 变量 | 说明 | 必填 |
|-----|------|------|
| `JWT_SECRET_KEY` | JWT签名密钥（`python -c "import secrets; print(secrets.token_urlsafe(64))"` 生成） | ✅ |
| `ENCRYPTION_KEY` | Fernet加密密钥（`generate_key.py` 生成） | ✅ |
| `FEISHU_APP_ID` | 飞书应用ID | ✅ |
| `FEISHU_APP_SECRET` | 飞书应用密钥 | ✅ |
| `FEISHU_VERIFICATION_TOKEN` | 飞书 Webhook 验签 Token（防止伪造事件） | ✅ |
| `FEISHU_ENCRYPT_KEY` | 飞书 Webhook 加密密钥 | ✅ |
| `SUPER_ADMIN_FEISHU_USER_ID` | 超管飞书用户ID（`ou_xxx` 格式，初始化超管账号必须配置） | ✅ |
| `ALLOWED_ORIGINS` | CORS允许源 | ✅ |
| `CLOUDFLARE_API_TOKEN` | Cloudflare API令牌 | 可选 |
| `GODADDY_API_KEY` | GoDaddy API Key | 可选 |
| `GODADDY_API_SECRET` | GoDaddy API Secret | 可选 |
| `DNSPOD_SECRET_ID` | DNSPod密钥ID | 可选 |
| `DNSPOD_SECRET_KEY` | DNSPod密钥Key | 可选 |

> **注意**：服务商 Token（Cloudflare/GoDaddy/DNSPod）按实际使用的服务商配置，未使用的留空即可。

生成密钥工具: `generate_key.py`（仅生成 `ENCRYPTION_KEY`，`JWT_SECRET_KEY` 需单独用 `secrets.token_urlsafe(64)` 生成）

### Docker部署

```bash
# 1. 复制环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env 填入真实配置（重点：JWT_SECRET_KEY、ENCRYPTION_KEY、飞书相关、SUPER_ADMIN_FEISHU_USER_ID）

# 2. 启动服务（注意：使用 Docker Compose V2，命令为 docker compose，无横线）
docker compose up -d

# 3. 查看日志
docker compose logs -f backend
docker compose logs -f frontend
```

> **首次部署 — 超管账号初始化**：确保 `.env` 中已配置 `SUPER_ADMIN_FEISHU_USER_ID`（填写超管的飞书 `ou_xxx` 用户ID），然后运行：
> ```bash
> docker compose exec backend python scripts/init_db.py
> ```
> init_db.py 会自动以该飞书ID创建超管账号。  
> ⚠️ `manage_users.py` 脚本无法添加 `super_admin` 角色（argparse 限制），首次超管账号**只能通过 init_db.py 创建**。

### SSL证书管理说明

**重要说明**: 本SSL证书管理功能针对**系统本身的Web访问域名**（例如 `d.fwxg.com`），**不是**用于管理系统所管理的业务域名的SSL证书。

文档位置: `docs/archive/ssl-certificate-management.md`

主要功能:
- **证书获取**: 使用 ACME 协议（Let's Encrypt）自动获取证书
- **证书监控**: 定时任务检查证书过期时间（剩余30天警告，7天紧急）
- **自动续期**: 剩余3天内自动发起续期
- **证书存储**: 使用 Certbot 管理，存储于 `/etc/letsencrypt/`
